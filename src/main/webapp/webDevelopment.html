Basics

What is Web Development?

Web development refers to building, creating, and maintaining websites and web applications — covering web design, publishing, programming, and database management. It includes both client-side (frontend) and server-side (backend) work.

Frontend

Client-side technologies

Frontend development builds what users see and interact with in their browser.

  • HTML — structure and content using elements, tags, and attributes.
  • CSS — presentation and layout; responsive design for different screens.
  • JavaScript — interactivity: DOM updates, events, animations, form validation.

Starter HTML template

<!doctype html>
<html>
  <head><meta charset="utf-8"/></head>
  <body><h1>Hello Web!</h1></body>
</html>

Quick CSS example

body {font-family: Arial, sans-serif; margin:0;}
.container{max-width:960px;margin:0 auto;padding:16px}
Backend

Server-side languages & responsibilities

The backend runs on servers and handles business logic, authentication, data access, and integration with external services.

  • Node.js (JavaScript) — event-driven, great for APIs and real-time apps.
  • PHP — traditional web scripting used in many CMS platforms.
  • Python — readable, great for web frameworks (Django, Flask).
  • Ruby — elegant syntax, Rails is popular for rapid development.
  • Java — enterprise-grade, scalable systems.
  • Go (Golang) — performant, compiled language for high-concurrency services.
  • C# — modern OOP, commonly used with .NET on Windows/Azure stacks.

Example: Simple Node.js HTTP server

const http = require('http');
const port = 3000;
http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type':'text/plain'});
  res.end('Hello from Node!');
}).listen(port);
console.log('Server running on port', port);
Databases

SQL vs NoSQL — choosing storage

Databases store and organize the data used by your application. Choose relational (SQL) when you need strict schemas and relationships; choose NoSQL when you need flexible schemas or horizontal scalability.

Relational (SQL)

  • MySQL — popular, reliable, widely hosted.
  • PostgreSQL — advanced features, strong consistency, extensible.

NoSQL

  • MongoDB — document store (JSON-like documents), flexible.
  • Cassandra — wide-column store for high write throughput and availability.
  • Redis — in-memory datastore for caching, sessions, real-time counters.
APIs

APIs & data formats

APIs let different parts of your system (or third-party services) communicate. Use JSON for most REST APIs; XML still appears in legacy systems and some integrations.

REST vs GraphQL

  • REST — resource-oriented, simple to cache and route.
  • GraphQL — flexible queries from the client, single endpoint.

Small JSON example

{
  "user": {"id": 1, "name": "Asha"}
}
Workflow

Version Control & Deployment

Version control (Git) is essential for team development. Deployment publishes your code to servers — today this is often automated via CI/CD pipelines that run tests and push releases.

  • Git — branching, pull requests, code reviews.
  • CI/CD — GitHub Actions, GitLab CI, Jenkins.
  • Hosting — cloud providers (AWS, GCP, Azure), platform services (Vercel, Netlify, Heroku).
Graphics

Canvas and SVG

Use <canvas> when you need pixel-level drawing and animations. Use SVG for scalable vector graphics, icons, and diagrams.
